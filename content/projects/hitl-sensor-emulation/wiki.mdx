## Purpose

The HITL Sensor Emulation Platform is a mixed-signal board intended to validate avionics sensor acquisition modules without relying on the final vehicle harness or physical sensors. The public version focuses on architecture, layout strategy, and validation planning rather than proprietary limits.

## Requirements

- Emulate RTD, current-loop pressure transducer, valve, thermocouple, and differential sensor interfaces.
- Keep analog emulation repeatable enough for automated regression tests.
- Provide safe bring-up access to rails, references, DAC outputs, ADC inputs, and actuator channels.
- Use Altium Designer exports as the source of portfolio artifacts.

## System Context

The board sits between a control computer and avionics acquisition hardware. A Raspberry Pi CM5-class controller commands DACs, load switches, and digital interfaces while ADC channels capture feedback for closed-loop validation.

## Architecture

The design separates the control, power, analog emulation, and actuator interface regions. This keeps fast digital edges and switching regulator currents away from the low-level analog front ends.

## Schematic Walkthrough

The top-level schematic is organized as functional sheets: power tree, RTD emulation, current-loop pressure transducer emulation, ADC front end, valve driver interface, and CM5 digital control.

### Analog Front End

The analog front-end section prioritizes reference stability, protected external interfaces, and measurable intermediate nodes. DAC-driven emulation paths include feedback measurement so firmware can detect drift or assembly faults.

## PCB Layout Walkthrough

The layout viewer callouts are manually defined in `project.json` with percentage coordinates. The top-layer placeholder highlights the buck converter hot loop, analog region, digital control area, and connector/test point placement.

Key layout decisions:

- Buck input capacitors and switching return paths are kept compact.
- Analog regions are placed away from high-current valve and harness areas.
- Test points are grouped by bring-up order.
- Digital control traces are routed with short, readable escape paths before entering mixed-signal areas.

## Bring-up Plan

1. Inspect board, connector polarity, and assembly options.
2. Power with a current-limited bench supply.
3. Verify 5 V, 3.3 V, and analog references.
4. Validate CM5 control communication.
5. Command DAC outputs without external loads.
6. Enable one emulation channel at a time.
7. Run automated sensor-response sweeps and capture evidence.
